Add Podman environment support Request Changes

Solid 1153-line Podman environment implementation that mirrors Docker's architecture. However, four inherited methods route through docker compose instead of podman-compose, resource limits are silently dropped, force_build is broken for prebuilt-image tasks, and environment directory files are not uploaded after start. All four bugs were reproduced at runtime.

What's Good

The overall architecture is clean — subclassing DockerEnvironment and overriding the core methods (exec, start, stop, attach) is the right approach. SELinux label handling is well-implemented with enforcing-mode detection. The container readiness polling with retry logic is thorough. Good test coverage for the basic happy path.

Recommendation

Fix the 4 reproduced bugs before merge: (1) override sidecar methods, (2) add resource limits compose file, (3) use should_use_prebuilt_docker_image(), (4) add environment dir upload call. Consider also adding temp file cleanup in stop().

Bug Inherited sidecar operations call docker compose instead of podman-compose bugs, correctness

PodmanEnvironment inherits service_exec, stop_service, and _compose_exec from DockerEnvironment. All of these route through _run_docker_compose_command, which invokes docker compose — not podman-compose.

Fix: Override stop_service and _compose_exec to route through _run_podman_compose_command.
